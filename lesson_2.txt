What happens when you initialize a repository? Why do you need to do it?

Creates a hidden directory structure with necessary files to work with git repo.

How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area is the files from the working directory that are being tracked and ready to be committed to the repo. It helps by allowing the user to group files into one commit, while leaving other files separate.

How can you use the staging area to make sure you have one commit per logical change?

Send changes with same logic to staging area, then commit. Then send different types of changes, etc.

What are some situations when branches would be helpful in keeping your history organized? How would branches help?

Different languages, different versions of a game, testing out a new idea you aren't sure of.

How do the diagrams help you visualize the branch structure?

easier to see the actual format than just a line of commits
